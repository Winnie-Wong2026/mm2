## 3. 主控决策

- `rank` 按“同策略、同市场内排名”作为第一版标准，便于分别输出 A股 Top N 和港股 Top N。
- `PyYAML` 第一版暂不强制作为硬依赖；策略注册器可先保留兜底解析器，后续依赖统一时再决定是否加入。
- `quality_score` 暂作为综合质量因子名，等待数据线程确认后再固化。
- Python 依赖管理统一使用 `pyproject.toml`；安装工具优先 `uv`，没有 `uv` 时用 `pip` fallback。
- 港股交易日历 MVP 阶段先允许用实际行情日期反推，后续找到稳定免费源后再独立接入。
- 内部标准化后的 `volume` 统一为股数；数据管道负责处理 A股“手”到“股”的换算，并保留必要的来源说明。
- 财务/估值第一版先接估值快照，详细财报字段进入后续迭代。
- 第一轮后端登录继续使用 mock token；进入内部试用前升级为真实 JWT。
- 观察名单和报告第一版使用 SQLite 持久化，当前 mock API 先保持接口不变。
- 任务状态由 `backend/tasks` 统一对外提供；数据线程负责输出数据更新状态，后端任务层负责聚合展示。
- 风险等级固定为四档：`低`、`中`、`中高`、`高`。
- 股票详情专业因子字段第一版按 `trend`、`liquidity`、`valuation`、`quality`、`volatility` 对齐。
- 首页第一版增加回测中心入口；回测线程未完成前可显示为基础入口或“建设中”状态。
- 观察名单备注第一版需要账号同步，先由后端 mock/SQLite 承接。
- 策略解释文案统一由 `backend/strategies/explanations.py` 提供，前端和 API 不重复拼接策略专属解释。
