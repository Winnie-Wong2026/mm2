# AI量化选股平台 PRD 与实施计划

## 1. 项目定位

本平台是一个面向小白用户的 AI 量化选股研究平台，主要覆盖 A股和港股主流股票池。平台以日频、周频收盘后更新为核心节奏，暂时不做交易执行，不接券商下单，只提供内部研究、候选股筛选、风险提示和解释型报告。

第一版采用本地部署，先使用免费数据源验证完整闭环。后续如需要多人远程访问、商业化或更稳定的数据质量，再升级部署方式和数据源。

## 2. 已确认需求

| 项目 | 当前结论 |
| --- | --- |
| 市场范围 | A股 + 港股 |
| 更新频率 | 日频、周频，收盘后更新 |
| 交易执行 | 暂时不做交易 |
| A股范围 | 主流股票，包含沪深主板、创业板、科创板，过滤 ST、停牌、低流动性等 |
| 港股范围 | 优先港股通、恒生综合、恒生指数/国企指数相关主流股票 |
| 持仓/观察周期 | 5日、10日、20日、4周、8周、12周都保留 |
| 每期输出数量 | A股 Top 20 / Top 50，港股 Top 20 / Top 50 |
| 数据策略 | 第一版先用免费数据 |
| 历史回测 | A股建议从 2015 年开始；港股按免费数据稳定区间确定 |
| 用户体系 | 需要考虑多人登录 |
| 报告能力 | 需要每日/每周中文解释报告 |
| 合规边界 | 内部研究，不对外收费 |
| 用户体验 | 小白友好 + 专业可展开 |

## 3. 产品原则

1. 不把平台做成复杂量化终端，首页优先展示结论、理由和风险。
2. 避免使用“买入、卖出、荐股”等强交易导向表达，统一使用“候选股、观察名单、研究参考、风险提示”。
3. 对小白用户先展示可理解语言，例如“趋势较强、成交活跃、估值不算贵、波动偏高”。
4. 专业指标仍然保留，但放在详情页或展开区域，例如因子分数、回测曲线、胜率、最大回撤、换手率、行业暴露。
5. 所有模型结果必须配合风险提示，避免用户误解为确定性结论。

## 4. 核心用户

### 4.1 小白用户

目标是快速看懂：

- 今天/本周哪些股票值得观察
- 为什么这些股票上榜
- 风险高不高
- 适合观察多久
- 是否可以加入自己的观察名单

### 4.2 研究人员/管理员

目标是进一步分析：

- 模型和因子是否有效
- 历史回测表现如何
- 哪些行业或风格暴露过高
- 数据是否更新成功
- 哪些股票被过滤，过滤原因是什么

## 5. 第一版功能模块

### 5.1 登录与权限

- 本地部署下也预留账号体系
- 角色分为管理员、研究员、普通用户
- 普通用户可查看榜单、股票详情、报告、自选观察名单
- 研究员可查看模型解释、因子详情、回测结果
- 管理员可查看数据更新状态、任务日志、用户管理

### 5.2 首页

首页面向小白用户，减少复杂表格，重点展示：

- 今日 AI 精选
- 本周 AI 精选
- A股 Top 20 / Top 50
- 港股 Top 20 / Top 50
- 市场整体状态
- 行业热度
- 风险提示
- 最新报告入口

每只股票卡片或列表项建议展示：

- 股票名称与代码
- 市场：A股 / 港股
- AI 综合评分
- 风险等级：低 / 中 / 中高 / 高
- 适合观察周期
- 入选理由摘要
- 加入观察名单按钮

### 5.3 股票详情页

详情页分为小白层和专业层。

小白层：

- 为什么上榜
- 当前优势
- 当前风险
- 适合观察周期
- 最近表现
- 行业位置
- 是否连续多期上榜

专业层：

- 因子明细
- 价格/成交额走势
- 行业对比
- 历史入选表现
- 模型贡献解释

### 5.4 观察名单

- 用户可收藏股票
- 支持备注
- 支持按 A股/港股、风险等级、上榜次数筛选
- 支持查看观察股票最近是否仍在榜单中

### 5.5 回测中心

第一版重点展示可理解的核心指标：

- 累计收益曲线
- 最大回撤
- 胜率

- 换手率

专业指标可放在展开区域：

- 本期市场概览

- 上期榜单变化
- 风险提示
- 数据更新时间

## 6. 数据流程

第一版采用免费数据源，建议优先用 AkShare 做原型。

数据流程：

1. 收盘后触发数据更新
2. 拉取 A股、港股行情数据
3. 拉取指数、行业、基础财务、估值等可用数据
4. 统一股票代码、交易日、币种和字段名称
5. 数据质量检查
6. 写入本地存储
7. 计算因子
8. 生成模型输入数据

需要做的数据检查：

- 是否缺少最新交易日
- 是否存在重复数据
- 是否存在异常价格
- 是否存在成交额为 0 的异常记录
- 是否复权数据不一致
- 是否股票状态异常

## 7. 股票池过滤规则

A股过滤：

- ST / *ST
- 停牌
- 上市时间少于 120 个交易日
- 最近 20 日平均成交额过低
- 长期无成交
- 极端异常价格
- 当日涨跌停导致不可交易的样本，在回测中单独处理

港股过滤：

- 长期停牌
- 成交额过低
- 极端低价股
- 财务数据严重缺失
- 波动异常且流动性不足

## 8. 因子体系

第一版先做基础因子，保证可解释和可回测。

### 8.1 趋势动量

- 5日收益率
- 20日收益率
- 60日收益率
- 相对行业收益
- 相对指数收益

小白解释：近期趋势是否强。

### 8.2 短期反转

- 1日反转
- 5日反转
- 过热回落风险

小白解释：短期是否涨太快，是否有回调风险。

### 8.3 成交量与流动性

- 20日平均成交额
- 成交额变化率
- 换手率
- 量价配合度

小白解释：资金活跃度是否提升。

### 8.4 波动与风险

- 20日波动率
- 60日波动率
- 最大回撤
- 下跌波动

小白解释：价格波动是否过大。

### 8.5 估值与财务质量

- 市盈率
- 市净率
- ROE
- 营收增速
- 净利润增速
- 毛利率

小白解释：公司是否便宜、质量是否稳。

## 9. 模型方案

第一版建议使用 LightGBM 排序/回归模型。

日频目标：

- 未来 5 个交易日相对收益
- 未来 10 个交易日相对收益
- 未来 20 个交易日相对收益

周频目标：

- 未来 4 周相对收益
- 未来 8 周相对收益
- 未来 12 周相对收益

模型输出：

- 综合评分
- 市场内排名
- 行业内排名
- 风险等级
- 入选理由
- 主要正向因子
- 主要负向因子

## 10. 回测规则

回测必须从第一版就严格处理：

- 禁止使用未来数据
- 财务数据按披露日可见，不能按报告期直接穿越
- 过滤生存者偏差
- 加入手续费
- 加入滑点
- 加入涨跌停限制
- 加入停牌处理
- 加入成交额容量限制
- 记录换手率

建议基准：

- A股：沪深300、中证500、中证全指
- 港股：恒生指数、恒生综合指数、恒生中国企业指数

### 11.3 前端

### 11.5 策略扩展架构

平台从第一版开始就要按“多策略平台”设计，不能把日频榜单、周频榜单、LightGBM 模型、因子规则直接写死在主流程里。后续新增策略时，理想状态是新增一个策略目录和一份策略配置，就能接入数据、回测、打分、榜单和报告。

建议把系统拆成以下稳定模块：

- 数据层：只负责获取、清洗、存储行情、财务、行业、指数、交易日历等基础数据
- 股票池层：统一生成可交易股票池，例如 A股主流池、港股主流池、行业池、自定义观察池
- 因子层：每个因子独立实现，输出标准化因子表，不绑定具体策略
- 策略层：每个策略声明自己需要的股票池、因子、调仓频率、持有周期、打分方式和风控规则
- 回测层：接收标准化策略信号，统一处理手续费、滑点、停牌、涨跌停、成交额约束
- 评分层：把不同策略输出转换为统一的综合评分、排名、风险等级和解释字段
- 报告层：根据统一结果生成日报、周报和股票详情解释
- 展示层：前端只读取统一榜单和解释结果，不直接依赖某个具体策略

每个策略建议包含：

- strategy_id：策略唯一标识
- name：策略名称
- market：适用市场，例如 A股、港股、全部
- frequency：日频或周频
- universe：使用哪个股票池
- factors：使用哪些因子
- model：规则模型、LightGBM、其他模型
- horizon：观察/持有周期
- rebalance_rule：调仓规则
- risk_rule：过滤和风控规则
- explanation_template：小白解释模板

建议的策略目录结构：

```text
strategies/
  momentum_quality_daily/
    strategy.yaml
    signal.py
    explain.py
  low_vol_value_weekly/
    strategy.yaml
    signal.py
    explain.py
```

新增策略时应尽量只做三件事：

1. 新增策略配置，声明市场、频率、股票池、因子和模型
2. 新增信号生成逻辑，输出统一格式的股票评分
3. 新增解释模板，把专业指标翻译成小白能理解的说明

主流程不应该因为新增策略而频繁修改。数据更新、因子计算、回测、榜单生成、报告生成都应该通过统一接口调用策略。

## 14. 待进一步确认

以下问题不影响 MVP 启动，但会影响后续设计：

1. 本地部署是在一台 Mac 上使用，还是局域网内多人访问？
2. 多人登录第一版是否需要密码找回、用户注册，还是管理员手动建账号即可？
3. 每日/每周报告是否需要导出 PDF/Excel？
4. 是否需要手机端适配？
5. 是否需要中文大模型生成更自然的解释文案？
6. 后续是否可能接入付费数据源，例如 Tushare Pro、Wind、聚宽、米筐等？

## 15. 第一版成功标准

第一版不以“收益率很高”为唯一成功标准，而以完整闭环和可信度为标准：

- 能稳定更新 A股和港股主流股票数据
- 能每天/每周生成候选榜单
- 每只候选股都有小白可理解的解释和风险提示
- 回测逻辑没有明显未来函数和数据穿越
- 用户可以登录、收藏、查看报告
- 管理员可以查看数据更新时间和任务状态
- 平台可以在本地稳定运行
- 新增一个策略时，不需要改动数据更新、回测、榜单和报告主流程
