# 架构总览

## 1. 当前阶段目标

第一阶段目标是搭建一个本地可运行、方便扩展策略的 AI 量化选股平台骨架。平台覆盖 A股和港股主流股票池，支持日频和周频收盘后更新，暂不做交易执行。

## 2. 模块边界

| 模块 | 目录 | 职责 |
| --- | --- | --- |
| 数据源 | `backend/data_sources/` | 接入 AkShare 等免费数据源 |
| 数据管道 | `backend/data_pipeline/` | 清洗、标准化、存储、质量检查 |
| 因子 | `backend/factors/` | 独立计算可复用因子 |
| 策略 | `backend/strategies/` | 插件化策略配置、宏观门槛、板块/仓位约束、信号生成 |
| 评分 | `backend/scoring/` | 把策略信号转成榜单、评分、风险等级 |
| API | `backend/api/` | FastAPI 路由和接口 |
| 权限 | `backend/auth/` | 登录、用户、角色 |
| 任务 | `backend/tasks/` | 收盘后更新、任务状态、日志 |
| 前端 | `frontend/` | 小白友好的页面体验 |
| 数据文件 | `data/` | 本地原始数据和处理后数据 |

## 3. 核心设计原则

1. 数据、因子、策略、回测、展示分层，不互相写死。
2. 新增策略时，优先新增策略目录和配置，不改主流程。
3. 前端只读取统一 API，不直接依赖某个策略文件。
4. 回测只接收标准化策略信号，不关心信号来自规则策略还是 AI 模型。
5. 面向小白用户的解释字段必须从后端统一输出，避免前端自行拼接专业逻辑。
6. 策略执行顺序固定为“宏观情况 -> 板块方向 -> 仓位配置 -> 风险提醒 -> 个股筛选”；大方向不通过时不生成个股榜单。

## 4. 第一阶段并行线程

- 数据线程：负责免费数据源和数据管道。
- 策略框架线程：负责策略插件、因子接口和统一信号格式。
- 前端体验线程：负责 mock 数据页面原型。
- 后端平台线程：负责 API、登录、任务状态和 mock 接口。

跨模块接口变更必须回到主控对话确认。
