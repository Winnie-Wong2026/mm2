# 数据源评估

## 1. 评估范围

本轮只评估第一版本地部署可用的免费数据源，重点看 A股和港股主流股票池的日频/周频收盘后数据闭环。第一版不做交易执行，也不依赖实时盘口。

当前建议：第一版以 AkShare 作为原型数据入口，但必须把数据源接口封装在 `backend/data_sources/`，不要让策略、因子、评分或前端直接依赖 AkShare 的函数名和中文字段。

## 2. AkShare 可用性结论

| 维度 | 结论 | 第一版建议 |
| --- | --- | --- |
| A股日线行情 | 可用。官方示例包含 `stock_zh_a_hist`，支持日频、前复权、后复权和不复权。 | 用作 A股日线主入口，默认保存不复权数据；前复权数据可作为因子计算视图。 |
| A股实时/收盘快照 | 可用。官方文档有沪 A、深 A、京 A 以及全 A 相关实时行情接口，字段含价格、涨跌幅、成交量、成交额、换手率、市盈率、市净率等。 | 用于股票池、估值快照和收盘后状态检查，不作为交易执行依据。 |
| 港股日线行情 | 可用。官方文档有 `stock_hk_hist`，示例可返回港股日频历史行情，字段含日期、开高低收、成交量、成交额、涨跌幅、换手率等。 | 用作港股日线主入口；先覆盖港股通和核心指数成分，暂不抓全市场。 |
| 港股实时/收盘快照 | 可用但有延迟和字段差异。港股通实时行情文档明确存在延迟/单位差异，港股分钟接口也提示港股有延时。 | 只用于收盘后补充，不在第一版展示为实时行情。 |
| 交易日历 | A股交易日历可用，`tool_trade_date_hist_sina` 提供股票交易日数据。 | 第一版可作为 A股日频调度校验；港股交易日历需另做补充或用实际行情日期反推。 |
| 财务/估值 | A股、港股均有若干财务接口，但字段宽、口径复杂、披露日穿越风险高。 | MVP 只接入估值快照和少量质量字段；财务因子等回测线程确认披露日规则后再扩大。 |
| 稳定性 | 免费接口依赖第三方网页/接口，字段名和可用性可能变化。 | 必须保留原始数据、字段映射、质量检查和本地缓存；失败时使用上次可用数据并标记数据状态。 |

## 3. 推荐接口清单

| 数据类型 | 市场 | AkShare 接口 | 用途 | 注意事项 |
| --- | --- | --- | --- | --- |
| 股票快照/股票池 | A股 | `stock_zh_a_spot_em` | 获取 A股主列表、价格、成交额、换手率、估值快照 | 字段为中文；成交量单位需实测确认并在标准化层记录。 |
| 日线行情 | A股 | `stock_zh_a_hist` | 获取单只 A股日线行情 | `adjust=""`、`qfq`、`hfq` 都应保留参数，不在策略里写死。 |
| 港股快照/股票池 | 港股 | `stock_hk_spot_em` | 获取港股主列表和收盘后快照 | 免费港股行情可能延迟，第一版不承诺实时性。 |
| 港股通池 | 港股 | `stock_hsgt_sh_hk_spot_em` | 生成港股通优先股票池 | 官方文档确认沪港通下的港股通接口；深港通拆分接口需实测后再固化。 |
| 日线行情 | 港股 | `stock_hk_hist` | 获取单只港股日线行情 | 港股代码统一补齐 5 位，例如 `00700.HK`。 |
| A+H 参考 | A股/港股 | `stock_zh_ah_spot_em` | A+H 比价、跨市场参考 | 文档标注延迟 15 分钟，只做研究参考。 |
| 交易日历 | A股 | `tool_trade_date_hist_sina` | A股交易日校验 | 免费日历不等同港股/港股通交易日历。 |
| 指数日线 | A股 | `index_zh_a_hist` | 沪深 300、中证 500、上证指数等基准行情 | MVP 先接 A股指数日线；港股指数历史源另行确认。 |
| 估值快照 | A股/港股 | `stock_zh_a_spot_em`、`stock_hk_spot_em` | 获取市盈率、市净率、市值、成交额等快照字段 | 快照字段用于过滤和解释辅助，不直接等同经披露日处理后的财报因子。 |
| 港股财务 | 港股 | `stock_financial_hk_report_em` | 港股三大报表原始数据 | 不直接进模型；需披露日和字段口径确认。 |

## 4. 第一版数据策略

1. A股主流程：`stock_zh_a_spot_em` 生成候选股票池，逐只调用 `stock_zh_a_hist` 拉取日线。
2. 港股主流程：优先用港股通池加核心指数成分作为主流港股池，逐只调用 `stock_hk_hist` 拉取日线。
3. 原始数据全部落到 `data/raw/`，处理后统一字段落到 `data/processed/`。
4. 每次数据更新都写入 `source`、`source_function`、`ingested_at`、`adjust_type`、`currency`、`amount_currency` 等来源字段。
5. 策略和因子只读取标准化后的 `daily_bars`、`securities` 等内部表，不读取 AkShare 原始中文字段。
6. 免费数据失败时不静默吞掉，数据管道必须返回质量检查结果和错误摘要，供任务状态线程展示。
7. 交易日历、指数行情和估值快照同样走 `raw/` 到 `processed/` 的双层落盘，不直接给策略层暴露原始接口字段。

## 5. 主要限制和风险

| 风险 | 影响 | 应对 |
| --- | --- | --- |
| 免费接口字段变化 | 标准化失败或空数据 | 字段映射集中放在 `backend/data_pipeline/normalizers.py`，质量检查发现缺字段即失败。 |
| 港股行情延迟 | 当日收盘后数据可能滞后 | 港股日线更新增加重试窗口，并展示 `latest_trade_date`。 |
| 港股交易日历缺口 | 周频聚合、缺失检查可能误判 | MVP 先用实际日线日期校验港股，后续主控确认独立港股日历源。 |
| 财务数据披露日穿越 | 回测结果失真 | 第一版不把财务报表直接作为训练标签或因子输入，先记录原始表和披露时间需求。 |
| 复权口径差异 | 因子、回测口径混乱 | `daily_bars` 增加 `adjust_type`，不同复权数据作为不同数据视图保存。 |
| 单只股票循环慢 | 全量更新耗时长 | 第一版先小股票池验证，后续可加断点续跑、并发限速和本地缓存。 |
| 港股指数历史源未固化 | 港股相对指数收益和基准回测暂不完整 | MVP 先用港股个股行情闭环；港股指数历史源由主控确认后补充。 |

## 7. 外部数据源容错

实测时，东财部分接口会出现连接失败或远端断开的情况。当前 Provider 已增加备用源：

| 数据 | 主接口 | 备用接口 |
| --- | --- | --- |
| A股日线 | `stock_zh_a_hist` | `stock_zh_a_daily` |
| 港股日线 | `stock_hk_hist` | `stock_hk_daily` |
| A股指数日线 | `index_zh_a_hist` | `stock_zh_index_daily_tx` |
| A股快照 | `stock_zh_a_spot_em` | `stock_zh_a_spot` |
| 港股快照 | `stock_hk_spot_em` | `stock_hk_spot` |

备用快照源可保证股票池、价格、成交额等字段落盘，但 PE/PB/总市值字段可能缺失，质量检查会用 `missing_valuation_metrics` 警告标出。

## 8. 当前环境检查

本机系统 `python3` 为 3.9.6，但项目已通过 `uv` 创建 `.venv`，实际项目环境为 Python 3.11.15，并已安装 `akshare`、`pandas`、`duckdb`、`pyarrow` 等依赖。数据线程仍保留延迟导入：

- 没有 AkShare 时，可以导入数据管道模块并单测字段标准化、质量检查、本地 JSONL 存储。
- 使用 `python3 -m uv run python scripts/run_live_data_sample.py` 可重新执行核心行情真实数据小样本拉取。
- 使用 `python3 -m uv run python scripts/run_live_data_sample.py --include-snapshots` 可额外拉取全市场快照；该接口较慢且外部源波动更明显。
- 不建议直接用系统 `python3` 运行项目，因为它仍是 3.9.6。
