# 页面原型说明

## 1. 原型范围

本轮前端原型按两层信息架构组织：第一层给小白用户看，第二层给研究人员看。写入范围为 `frontend/` 与本说明文档。原型不接真实数据源、不实现策略模型、不修改后端接口代码。

当前前端采用无外部依赖的静态单页应用，原因是当前环境可用 Node，但没有可用包管理器。运行方式为：

```bash
cd frontend
node server.mjs
```

然后访问 `http://localhost:5173`。

## 2. 信息架构

### 2.1 首页

首页作为第一层入口，用于快速回答小白用户最关心的问题：

- 当前宏观环境是否允许进入个股筛选
- 应优先观察哪些板块/行业
- 当前建议仓位和单只上限是什么
- 大方向风险提醒是什么
- 今天有哪些候选股值得观察
- 本周有哪些候选股持续靠前
- AI 综合评分是多少
- 风险等级是什么
- 为什么上榜
- 适合观察多久
- 主要风险是什么
- 是否进入观察名单
- 是否需要查看 K 线明细
- 当前使用哪套选股策略，是否需要更换策略

首页改为三段式工作台，避免把所有说明和股票卡片堆在同一屏：

1. 今日决策概览：只展示宏观状态、当前策略、仓位上限和下一步入口。
2. 候选股工作台：只预览今日精选和本周精选 Top 3，完整列表进入对应页面。
3. 研究入口：只提示因子、回测、收益曲线等字段在第二层查看。

板块方向和风险提醒放在候选股工作台右侧，作为筛选前的背景信息，不再占用整屏。

首页首屏不展示因子、回测、收益曲线等研究指标。研究人员内容统一放到第二层“研究中心”或股票详情的“研究人员视图”展开区。

### 2.2 今日精选

今日精选展示日频收盘后更新的候选股，强调短中期变化。页面字段包括：

- AI 综合评分
- 风险等级
- 为什么上榜
- 适合观察多久
- 主要风险是什么
- 是否进入观察名单
- 查看 K 线

K 线通过股票卡片上的“查看K线”按钮打开弹窗展示。弹窗包含股票基础信息、AI 综合评分、风险等级、入选理由、观察周期、主要风险和 K 线图。

当前 K 线为 mock OHLC 数据，支持：

- 日K / 周K / 月K 切换
- 近1月 / 近3月 / 近6月 / 近1年筛选
- 中国市场习惯：红涨、绿跌

策略更换入口放在页面顶部工具区，作为全局选择器常驻展示。原因是策略会影响今日精选、本周精选、A股榜单、港股榜单和股票详情中的评分展示，不适合藏在单个股票卡片或研究中心里。

首页右侧同步展示当前策略摘要，包括：

- 策略名称
- 风险偏好
- 观察周期
- 覆盖市场
- 主要因子方向

### 2.3 本周精选

本周精选展示周频候选池，强调连续性和复盘价值。页面结构与今日精选一致，但排序使用周频 mock 排名。

### 2.4 第二层：研究人员视图

研究人员视图集中展示专业分析字段：

- 因子明细
- 历史回测
- 收益曲线
- 最大回撤
- 胜率
- 换手率
- 行业暴露
- 模型解释

第二层入口包括：

- 侧边导航“研究中心”
- 股票详情页中的“展开研究人员视图”
- A股榜单、港股榜单、报告中心等研究/复盘页面

前端只展示 mock 或后端返回的研究指标，不在前端自行计算策略逻辑。

### 2.5 A股榜单与港股榜单

榜单页归入第二层研究视图，采用表格化展示，适合快速比较。当前支持：

- `Top 20 / Top 50` 切换
- 日频 / 周频切换
- 风险等级筛选

后续真实版本需要继续补充：

- 行业筛选
- 上榜次数筛选
- 策略版本筛选

### 2.6 股票详情

股票详情分为小白层和研究层：

- 小白层默认展示：AI 综合评分、风险等级、为什么上榜、适合观察多久、主要风险是什么、是否进入观察名单。
- 研究层通过展开按钮展示：因子明细、历史回测、收益曲线、最大回撤、胜率、换手率、行业暴露、模型解释。

前端只展示后端给出的解释和指标，不在前端自行推导策略逻辑。

### 2.7 观察名单

观察名单用于收藏候选股并记录观察理由。当前静态原型使用浏览器本地存储模拟；第一版集成目标需要接入用户账号，由后端 mock/SQLite 的 `GET /api/watchlist`、`POST /api/watchlist`、`DELETE /api/watchlist/{symbol}` 承接备注同步。

### 2.8 报告中心

报告中心归入第二层研究视图，展示每日、每周和专题报告的入口。报告内容原则是先给中文解释，再允许展开专业指标。

### 2.9 研究中心

研究中心是第二层主入口。当前页面展示指标框架和样本区间提示，包括因子明细、历史回测、收益曲线、最大回撤、胜率、换手率、行业暴露和模型解释。

回测结果、曲线数据、基准口径和接口路径等待回测线程确认；前端只负责展示，不自行计算研究指标。

## 3. 页面文案原则

1. 使用“候选股、观察名单、研究参考、风险提示、适合观察周期”等表达。
2. 首屏不使用强交易导向措辞，避免让小白用户误解为确定性结论。
3. 每个候选股展示评分时必须同时展示风险等级。
4. 专业指标不删除，但默认收进第二层，避免第一屏压迫感过强。
5. 所有趋势、评分、回测和报告样例都标注为 mock 或研究参考。

## 4. Mock 数据结构

前端 mock 数据位于 `frontend/src/mockData.js`，主要包含：

- `marketSummary`：首页市场状态、更新时间、行业热度。
- `macroRegime`：宏观门槛、板块方向、仓位配置和风险提醒。
- `strategyProfiles`：当前可切换的策略样例、风险偏好、观察周期、覆盖市场、因子权重和接口参数。
- `backtestSummary`：研究中心占位指标、样本区间和后续对接提示。
- `researchProfiles`：第二层研究人员视图 mock 数据。
- `stocks`：A股和港股候选股样例。
- `reports`：报告中心样例。
- `apiContracts`：后续对接 API 的页面映射。

候选股字段包括：

- `name` / `symbol` / `market` / `board`
- `score` / `risk` / `horizon`
- `dailyRank` / `weeklyRank` / `consecutive`
- `summary` / `strengths` / `risks`
- `factors` / `priceSeries`
- mock K 线由前端基于候选股样例生成，正式版本应由行情接口返回 OHLCV

研究人员视图字段包括：

- `backtestPeriod`
- `cumulativeReturn`
- `maxDrawdown`
- `winRate`
- `turnover`
- `equityCurve`
- `industryExposure`
- `modelExplanation`

## 5. 后续 API 对接点

| 页面 | API 草案 | 前端用途 |
| --- | --- | --- |
| 策略选择 | `GET /api/strategies` | 获取可用策略、策略说明、风险偏好、适用市场和默认策略 |
| 宏观状态 | `GET /api/macro-regime` | 获取宏观门槛、板块方向、仓位建议和风险提醒 |
| 首页状态 | `GET /api/tasks/status` | 数据更新时间、榜单生成时间、报告生成状态 |
| 今日精选 / 本周精选 / 榜单 | `GET /api/rankings` | 按市场、频率、Top N、`strategy_id` 获取候选股 |
| 股票详情 | `GET /api/stocks/{symbol}` | 获取解释、风险、因子摘要、走势和行业对比 |
| K 线弹窗 | 待行情接口确认 | 获取股票 OHLCV，支持日K / 周K / 月K 和时间区间筛选 |
| 观察名单 | `GET /api/watchlist` / `POST /api/watchlist` / `DELETE /api/watchlist/{symbol}` | 获取、添加、移出用户观察对象 |
| 报告中心 | `GET /api/reports` / `GET /api/reports/{report_id}` | 获取报告列表和报告正文 |
| 研究中心 | 待回测线程确认 | 获取因子明细、历史回测、收益曲线、最大回撤、胜率、换手率、行业暴露和模型解释 |

## 6. 已同步的主控决策与遗留边界

1. 风险等级固定为 `低 / 中 / 中高 / 高`，前端 mock 数据已补齐四档样例。
2. 首页第一版只展示小白层信息；研究指标统一收进第二层“研究中心”和详情页展开区。
3. 观察名单备注第一版需要账号同步；当前静态原型仍用 `localStorage` 模拟，集成时改接后端观察名单 API。
4. 榜单页默认展示 `Top 20`，并提供 `Top 20 / Top 50` 和日频 / 周频切换。
5. 股票详情专业因子字段第一版按 `trend / liquidity / valuation / quality / volatility` 对齐。
6. K 线颜色按中国市场习惯：红涨、绿跌。
7. 策略更换入口固定在顶部工具区，作为影响全局榜单和评分展示的入口；当前为 mock 策略，正式版需要后端返回可用策略和默认策略。
8. 策略输出前必须先展示宏观门槛、板块方向、仓位配置和风险提醒；大方向不通过时前端不展示个股榜单。
9. 首页布局收敛为“今日决策概览 / 候选股工作台 / 研究入口”三段，减少字段说明和完整卡片在首页重复出现。
10. 研究中心正式接口路径、曲线字段和基准口径仍等待回测线程输出后再固化。
